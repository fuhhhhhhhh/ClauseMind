## 2. AI 应用情况报告

### 2.1 使用的 AI 工具

| AI 工具 | 用途 | 使用比例 |
|---------|------|---------|
| **GitHub Copilot** | 代码生成、代码补全、架构设计、Bug 修复 | 70% |
| **ChatGPT / DeepSeek** | 技术方案讨论、代码 Review、优化建议 | 20% |
| **LLM API (审查 Agent)** | 系统运行时调用，分析合同文本 | 10% |

### 2.2 AI 在开发各阶段的应用

**（1）需求分析阶段**
- 使用 AI 分析课程设计要求，提取关键要点
- AI 辅助梳理功能清单和系统边界
- AI 生成竞品分析初稿，人工审核后定稿

**（2）系统设计阶段**
- AI 协助进行技术选型分析（FastAPI vs Spring Boot）
- AI 生成 UML 图（类图、时序图、组件图）
- AI 辅助数据库表结构设计
- AI 生成 API 接口定义

**（3）编码实现阶段**
- **后端核心代码**：Agent 框架、LLM Client、Normalizer 等核心逻辑由 AI 辅助生成
- **前端页面**：90% 的 React 组件代码由 Copilot 生成
- **API 路由**：AI 根据接口定义自动生成 FastAPI 路由代码
- **测试代码**：AI 生成 pytest 测试用例
- **数据库迁移**：AI 辅助编写 Alembic migration 脚本

**（4）调试测试阶段**
- AI 帮助分析错误日志，提供修复建议
- AI 辅助编写测试数据（seed_demo.py）
- AI 协助排查 bcrypt 兼容性等环境问题

**（5）文档编写阶段**
- AI 生成需求文档、设计文档、API 文档的初稿
- 人工核对文档内容与实现的一致性
- AI 辅助生成 Mermaid 图表代码

### 2.3 AI 应用经验总结

| 方面 | 经验 |
|------|------|
| **有效做法** | 1. 将复杂任务拆分为小步骤，逐步让 AI 实现<br>2. 先定义好接口和数据结构，再让 AI 填充实现<br>3. 让 AI 先生成测试，再根据测试实现功能 |
| **注意事项** | 1. AI 生成的代码需要人工审查，可能存在逻辑错误<br>2. AI 可能使用过时的 API 或库版本<br>3. AI 对项目全局理解有限，需要人工把控架构 |
| **改进空间** | 1. 可以使用更结构化的 prompt 模板<br>2. 建立项目规范文档让 AI 保持一致风格<br>3. 积累常见问题的解决方案库 |

### 3.4 主要开发活动详情

**重点活动 1：多 Agent 框架实现**

活动描述：设计并实现了 BaseAgent 抽象基类，包含 LLM 调用、JSON 解析、重试机制、提示词模板加载等通用功能。在此基础上实现了 6 个专用 Agent。

关键决策：
- 使用 `async/await` 实现 Agent 的异步执行
- JSON 解析失败时自动重试一次（携带更明确的指令）
- 每个 Agent 有独立的提示词模板文件（`prompts/*.txt`）

**重点活动 2：MinerU 集成**

活动描述：实现 `MinerUService` 通过 subprocess 调用 MinerU CLI，收集 Markdown、JSON、Layout PDF 等输出文件。

技术难点：
- MinerU 输出路径不固定，需要扫描目录自动发现
- 超时和错误处理需要包装为用户友好的中文提示
- GPU 环境配置需要透传 CUDA_VISIBLE_DEVICES

**重点活动 3：前端全流程实现**

活动描述：实现了从登录→上传→解析→审查→报告的完整前端流程，包含 12 个页面、8 个可复用组件、5 个 API 模块。

关键设计：
- Zustand 管理认证状态，localStorage 持久化
- Ant Design Steps 组件展示 Agent 审查进度
- react-markdown 渲染审查报告
- 路由守卫确保未登录用户无法进入受保护页面

## 4. 项目总结（可选）

### 4.1 成果回顾

ClauseMind 合同智能审查系统从零开始，在约一周的开发周期内完成了一个功能完整的、基于 React + FastAPI + MinerU + 多 Agent 的智能化系统。系统实现了：

- **后端**：10 个数据表、20+ REST API、6 个 AI Agent 工作流、完整的用户认证和权限控制
- **前端**：12 个功能页面、8 个可复用组件、前后端分离架构
- **文档**：需求/设计/API/数据库/UML 共 5 份文档
- **测试**：15 个测试文件，覆盖核心模块
- **总代码量**：约 7,000 行

### 4.2 技术亮点

1. **多 Agent 协作架构**：6 个专用 Agent 顺序执行，分工明确，可独立调试和演进
2. **端到端 AI 集成**：从文档解析（MinerU）到 LLM 分析（Agent），全流程 AI 驱动
3. **工程完整性**：Alembic 迁移管理、Pydantic 数据校验、JWT 鉴权、单元测试
4. **模块化设计**：核心功能模块清晰分离，便于扩展和维护

### 4.3 不足与改进方向

1. **LLM Agent 输出稳定性**：LLM 输出的 JSON 格式偶尔不符合预期，需要更强的提示词工程
2. **异步任务管理**：当前 Agent 审查是同步阻塞的，可以改为 BackgroundTasks 或 Celery
3. **前端测试**：缺少前端单元测试和 E2E 测试
4. **错误处理**：部分边界情况（如超大文件、网络断开）的处理可以更优雅
5. **多语言支持**：当前仅支持中文合同，可扩展支持英文和双语合同
